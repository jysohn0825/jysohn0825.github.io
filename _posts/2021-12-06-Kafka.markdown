### Kafka Intro

- kafka에 들어온 모든 데이터는 log 기반
- Offset을 통해 다음 consume 가능하게 확인 (offset 또한 로그로 기반 - 어디까지 읽었고 발행했는지 모두 로그로 확인)
- Topic, Consumer 마다 다르게 세팅 가능(어떤 건 배치, 어떤 건 실시간) -> Topic을 수신하는 수신자를 그룹별로 묶을 수 있다 (해당 세팅을 파일별로 묶을 수 있음)
- Rabbit은 유연함이 없어서 무조건 queue로 간다 (두 개의 차이가 있으므로 둘 다 사용해야만 한다)
- replication factor
메세지 발생 시 장애가 있을 수 있으므로 홀수의 복사본을 홀수로 만들어 놓음 -> ex) 4개 중 2개가 장애 발생 시 나머지 2개가 어떤 걸 가져갈지 모른다
복제본이 많다? = 복구가 어렵다
- 분산 서버
scale out 하기 좋음 (별도의 서버 추가 후 broker에 추가만 하면 알아서 분산을 해줌)
- 로그
I/O가 병목지점이다
파일을 I/O를 바로 하는게 아니라 OS 케시 페이지(메모리 영역)에 쓰기 떄문에 속도 보장
- 파티션을 늘린다(인스턴스를 늘리는 거와 같은 의미)
하나하나 쓰레드를 늘리는 것이기 때문에 너무 많이 x
추가는 언제든지 가능하나 줄이는 것은 불가능 -> L4세션 등의 개념이 없기 때문에 끊기는 것을 알 수 없으므로 줄일 수 없다
