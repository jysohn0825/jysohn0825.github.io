## 객체 지향 특징

- 추상화
- 캡슐화
- 상속
- 다형성


## 객체 지향 프로그래밍?

- 컴퓨터 프로그램의 객체 모임을 파악하기 위하 것
- 프로그램은 유연하고 변경에 용이하게 만들기 위한 것


## 다형성을 통한 역할과 구현의 분리

- 클라이언트는 대상의 역할(인터페이스)만 알면 된다
- 클라이언트는 구현 대상의 내부 구조를 몰라도 된다
- 클라이언트는 내부 구조가 변경되어도 영향을 받지 않는다
- 클라이언트는 대상 자체를 변경해도 영향을 받지 않는다

## 자바 언어에서 다형성

- 오버라이딩
- 다형성으로 인터페이스를 구현힌 객체를 실행 시점에 유연하게 변경
- 상속 관계에서도 가능


## 다형성의 본질

- 인터페이스를 구현한 객체 인스턴스르 실행 시점에 유연하게 변경
- 다형성의 본질을 이해하려먄 객체 사이의 협력을 알아야 함
- 클라이언트의 변경이 아닌, 서버의 기능을 변경해야한다


## 인터페이스 설계 주의 사항

- 역할이 변경 되면 서버,클라이언트의 너무 큰 변화가 생긴다


## 스프링과 객체 지향

- 스프링은 다형성을 극대화 한다
- IoC, DI는 다형성을 활용하여 역할과 구현을 편리하게 다룰 수 있도록 한다
