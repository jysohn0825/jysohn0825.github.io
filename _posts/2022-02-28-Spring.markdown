## 스프링의 종류

- 스프링 프레임워크
- 스프링 부트
- 스프링 데이터
- 스프링 배치
- 스프링 클라우드
- ...


## 스프링 프레임워크란?
- 핵심 : 스프링 DI 컨테이너, AOP, 이벤트, 기타
- 웹 기술 : MVC, WebFlux
- 데이터 : 트랜잭션, JDBC, ORM, ...
- 기술 : 캐시, 스케줄링, 원격, ...
- 테스트
- 언어 : 코틀린, 그루비


## 스프링 부트란?
- 스프링 프레임워크를 편리하게 사용할 수 있도록 지원
- 웹 서버를 내장하여 별도의 웹 서버 없이 단독으로 실행할 수 있는 스프링 애플리케이션 생성
- 쉬운 빌드 구성을 위한 starter 종속성 제공 (e.g. spring-boot-starter를 땡기면 필요한 많은 라이브러리를 땡김)
- 3rd parth 라이브러리와 스프링 버전에 맞게 자동 구성
- 프로덕션 준비 기능 (메트릭, 상태 확인, 외부 구성) 제공
- 관례에 의한 간결한 설정 (필요한 설정을 거의 다 디폴트로 해주기 때문에 필요에 따라 몇몇만 커스텀)


## 스프링이란?
- 스프링 DI 컨테이너 기술 (빈 관리 기술)
- 스프링 프레임워크
- 스프링 생태계(부트, 프레임워크 등등)


## why 스프링?
- 자바 언어 기반의 프레임 워크
- 자바 언어의 가장 큰 특징 : 객체 지향 언어
- 스프링은 객체 지향 언어가 가진 강력한 특징을 살려내는 프레임워크
- 스프링은 좋은 객체 지향 애플리케이션을 개발할 수 있게 도와주는 프레임워크 (DI, IoC 컨테이너를 통해)
